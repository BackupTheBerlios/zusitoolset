Zusi TCP-Datenausgabe Dokumentation

Version 1.1a, letztes Update: 1. Februar 2005

Changelog:
Version 1.1a: Hex-Werte der IDs hinzugefügt

Dieses Dokument erklärt das Zusi-TCP-Protokoll für die Datenausgabe. Mit Hilfe der Datenausgabe ist es möglich, interne Daten von Zusi für andere Programme zur Verfügung zu stellen. Diese können für Hilfsprogramme genutzt werden. Die Daten werden nahezu in Echtzeit übermittelt, was eine Anwendung für Fahrpulte ermöglicht.

Dieses Protokoll sowie die Anwendungen (sowohl Zusi selbst als auch der Verwaltungsserver und die Hilfsprogramme) sind ständigen Änderungen unterworfen. Sie sollten also in regelmäßigen Abständen nachsehen, ob Updates verfügbar sind. Dies gilt besonders dann, wenn Sie selbst Tools entwickeln.
Eine aktuelle Version dieses Dokumentes kann jederzeit unter http://www.smartcoder.net/eisenbahn/zusi/tcp heruntergeladen werden.
Zusi-Programmupdates können unter http://www.zusi.de/updates.htm heruntergeladen werden.

Das "Starterkit" enthält einige Tools, die für die Entwicklung von Nutzen sein können. Diese Tools werden ebenfalls in diesem Dokument beschrieben. Sie sind jedoch nur zum Testen und zur Fehlersuche nötig, für die finalen Programmversionen werden sie nicht benötigt.
Außerdem sind einige Codebeispiele speziell für Borland Delphi beigefügt, mit denen die größten Hürden leichter zu meistern sind. Für andere Programmiersprachen können die Lösungen entsprechend abweichen. Wer Lösungen für andere Sprachen ausgearbeitet oder Verbesserungsvorschläge hat, kann mir diese schicken, ich werde sie dann in die nächste Version des Dokuments einbauen.
