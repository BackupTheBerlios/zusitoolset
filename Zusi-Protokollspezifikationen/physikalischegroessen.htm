Übertragene Daten

Diese Größen entsprechen (zumeist) den Physikalischen Größen des Führerstandformats von Zusi. Sie werden mit dem Befehl DATA 0A (00 0A) als Single-Werte übertragen. Wie diese wieder in lesbare Formate umgewandelt werden, beschreibt der Abschnitt Programmierbeispiele: Single-Konvertierung. Wenn es sich um Boolsche Variablen handelt, steht 0 für False und 1 für True.
Einige Werte können nicht als Single übertragen werden (zum Beispiel Dateinamen). Diese werden als String übertragen. Bei den entsprechenden Werten steht der Typ String in Fettschrift daneben. Bei den Strings liefert das erste Byte die Längenangabe des Strings, danach folgt der eigentliche Wert. Die maximale Länge beträgt damit 255 Elemente. Ein leerer String wird durch die ID und die nachfolgende Länge 00 referenziert.
