00 02 - ACK_HELLO

Server » Client
Der Server teilt mit, ob der Client akzeptiert wird.

Länge Inhalt Bezeichnung Beschreibung
4 bytes xx xx xx xx PACKET_LENGTH Länge des nachfolgenden Datenpakets in Byte (Cardinal, Intel-Order)
2 bytes 00 02 ACK_HELLO Befehl ACK_HELLO
1 byte xx ACK Der Client wurde akzeptiert, wenn das Byte auf 0x00 steht. Wird der Client nicht akzeptiert, wenn zum Beispiel schon zu viele Clientverbindungen bestehen, wird stattdessen ein anderes Byte gesendet. Der Server bricht daraufhin die Verbindung ab.
Das Byte kann einen Fehlercode enthalten:
0x01: Zu viele Verbindungen
0x02: Zusi ist bereits verbunden, keine neue Verbindung mehr möglich
